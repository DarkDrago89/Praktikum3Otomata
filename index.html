📖 Teori PDA — Model Flowchart

Apa itu PDA Flowchart?

PDA (Pushdown Automata) adalah mesin komputasi yang memiliki memori tambahan berupa stack. Model flowchart menggambarkan alur kerja secara visual dengan node START, READ, PUSH, POP, ACCEPT, REJECT.

Tipe Node:

● START — Titik masuk, oval.
◆ READ — Diamond. Baca satu karakter. Cabang keluar berlabel kondisi (a/b/X/λ).
■ PUSH x — Kotak. Push simbol x ke stack.
■ POP — Kotak. Pop simbol dari top stack.
● ACCEPT — Oval ganda. Diterima jika input habis di sini.
● REJECT — Oval. Ditolak secara eksplisit.

Cara Kerja Stack

PUSH
Letakkan simbol di paling atas stack.
POP
Hapus dan ambil simbol dari top stack.
READ
Baca karakter, pilih cabang sesuai karakter.

Contoh: Palindrome sXsᴿ

String abXba diterima:

START → READ(a)→PUSH a → READ(b)→PUSH b → READ(X)
→ READ(b)→POP[b✓] → READ(a)→POP[a✓] → λ→POP[Z] → ACCEPT ✅

Tips Membuat PDA Flowchart

  • Mulai selalu dengan node START
  • Node READ harus punya edge untuk setiap kemungkinan karakter
  • Node PUSH dan POP hanya satu edge keluar
  • Gunakan λ untuk ε-transition (tidak konsumsi input)
  • Tambahkan ACCEPT di akhir jalur yang valid
